# マップ仕様書 v0.1

この文書は、戦略陣取りゲームの4人用マップ試作仕様書です。  
Codex などのコード生成エージェントに渡す前提で、見た目ではなく**ゲームロジック上のマップ**を定義します。

---

# 1. この仕様書の目的

この仕様書では、以下を固定します。

- 座標系
- 論理マップ
- 本拠地
- 中継拠点
- 湖
- 道
- 斜め道
- 兵団再編成エリア
- 橋候補
- 障害物設置条件
- 初期配置
- 描画方針

重要:  
この仕様書で扱うマップは、画像素材ではなく**論理マップ**です。  
isometric 表示や背景画像は、あとからこの論理マップを見た目として描画するだけであり、ゲーム判定には使いません。

---

# 2. 座標系

## 2.1 基本

- 左上を `(0,0)` とする。
- x は右方向に増える。
- y は下方向に増える。
- 斜め移動も1マスとして扱う。
- 距離判定は原則としてチェビシェフ距離を使う。

```text
distance = max(abs(dx), abs(dy))
```

## 2.2 マップサイズ

v0.1では仮に 25 × 25 の論理グリッドを使う。

- x: 0〜24
- y: 0〜24

このサイズは試作用であり、最終版では調整してよい。

# 3. 記号凡例

| 記号 | 意味 |
|---|---|
| `~` | 湖/水面 |
| `R` | 2マス幅の主道 |
| `D` | 1マス幅の斜め道 |
| `P` | 兵団再編成エリアとして使う追加道マス |
| `1` | チーム1本拠地 |
| `2` | チーム2本拠地 |
| `3` | チーム3本拠地 |
| `4` | チーム4本拠地 |
| `N` | 辺上の中立中継拠点 |
| `C` | 中央中立拠点 |

注意:

- `P` はルール上は道マスとして扱う。
- `D` もルール上は道マスとして扱う。
- `R`, `D`, `P` はすべて通常地上移動可能。
- `~` は通常駒では移動不可。忍者のみ水面移動可能。
- 橋はこの表には直接描かず、`BridgeSlot` として別定義する。

# 4. 4人用マップ v0.1 のコンセプト

## 4.1 拠点数

合計9拠点。

| 種別 | 数 |
|---|---:|
| 本拠地 | 4 |
| 辺上の中立中継拠点 | 4 |
| 中央中立拠点 | 1 |
| 合計 | 9 |

## 4.2 基本構造

- 4隅に各チーム本拠地を置く。
- 各チーム本拠地の間に、中立中継拠点を1つずつ置く。
- 盤面中央に中立中継拠点を1つ置く。
- 主道は十字路/外周連結に近い構造にする。
- 各本拠地から中央方面へ、1マス幅の斜め道を出す。
- 拠点周辺の道は各方面2マス幅にする。
- 斜め道は1マス幅とする。
- 道の中央付近に、兵団再編成用の追加マスを置く。

## 5.1 読み方

- これは見た目の完成図ではなく、ロジック用の仮マップである。
- 実際の描画では isometric 変換して表示する。
- `R`, `D`, `P` はすべて地上移動可能な道として扱う。
- `P` は隊列の詰まりを緩和するための再編成エリアであり、拠点ではない。
- `~` は湖であり、通常駒は移動できない。
- 忍者は `~` を水面として移動できる。
- 橋が設置された場合、橋セルは一時的に通行可能になる。

## 6.2 中立中継拠点

| id | 表記 | 種別 | 座標 | 初期所有 |
|---|---|---|---|---|
| `neutral_north` | `N` | neutral | `(11,3),(12,3),(11,4),(12,4)` | neutral_cpu |

| `neutral_center` | `C` | neutral | `(11,11),(12,11),(11,12),(12,12)` | neutral_cpu |

## 6.3 拠点共通ルール

- 拠点は見た目上2×2だが、戦闘上は内部リストで管理する。
- 通常拠点は最大4体まで駒を保持できる。
- 本拠地には奥座敷枠を1つ持たせる。
- 拠点内の駒は通常座標ではなく `BaseSlot` で管理する。
- 敵拠点へ入れるのは、その拠点内に敵駒が1体もいない場合のみ。
- 占拠優先権を持つチームのみが拠点を占拠できる。

# 7. 道定義

## 7.1 主道

`R` は2マス幅の主道である。

主道の役割:

- 本拠地と中立中継拠点をつなぐ。
- 中央中立拠点へ向かう主要経路を形成する。
- 2マス幅にすることで、拠点周辺の詰まりを減らす。

## 7.2 斜め道

`D` は各本拠地から中央方面に伸びる1マス幅の斜め道である。

斜め道の役割:

- 各本拠地から中央方面への短絡路を作る。
- ただし1マス幅なので、大軍の移動には向かない。
- 機動力のある駒や忍者上陸後の奇襲路として機能する。

## 7.3 兵団再編成エリア

`P` は兵団再編成用の追加道マスである。

- ルール上は道マスとして扱う。
- 生産・占拠・特殊支配は発生しない。
- 1マス幅の道や中央付近で隊列が詰まるのを緩和する。
- 拠点ではない。
- 中立拠点でもない。

---

# 8. 湖定義

## 8.1 湖の扱い

`~` は湖/水面である。

- 通常駒は移動できない。
- 忍者のみ水面移動可能。
- 水面上の忍者は攻撃不可。
- 水面上の忍者は兵種入れ替え不可。
- 水面上の忍者は拠点占拠不可。
- 忍者同士のみ水面上で戦闘可能。

## 8.2 lakeId

v0.1では、湖は便宜上4象限に分けて扱う。

| lakeId | 範囲イメージ |
|---|---|
| `lake_nw` | 左上象限の湖 |
| `lake_ne` | 右上象限の湖 |
| `lake_se` | 右下象限の湖 |
| `lake_sw` | 左下象限の湖 |

最終的には、連結成分として lakeId を自動計算してもよい。

# 9. 橋候補スロット

## 9.1 基本方針

橋は自由に任意座標へ生成しない。  
マップ側に定義された `BridgeSlot` から選択して設置する。

橋候補は以下の条件を満たす必要がある。

1. 同一湖内の直線区間である。
2. 縦または横一直線である。
3. 斜め橋は不可。
4. 始点側が自軍作戦圏の道マスに接している。
5. 終点側は同じ湖に接する別の道であればよい。
6. 終点側の道が自軍作戦圏である必要はない。

## 9.2 v0.1 仮BridgeSlot

# 10. 障害物設置条件

障害物は以下に設置できる。

- `R`
- `D`
- `P`
- 設置済みの橋上

障害物は以下に設置できない。

- 湖 `~`
- 拠点 `1`,`2`,`3`,`4`,`N`,`C`
- すでに駒がいるマス
- すでに障害物があるマス

障害物は自軍にも敵軍にも影響する。

# 12. 作戦圏

作戦圏は、軍師の橋・障害物・転送などの判定に使う。

## 12.1 定義

- 自軍拠点からつながる道マス。
- その道マスに面している湖。
- 障害物は作戦圏のつながりを遮断しない。
- 敵拠点/中立拠点は通過できない。
- ただし、敵拠点を挟んだ先にも別の自軍拠点がある場合、その先の道は別起点の作戦圏として扱う。

## 12.2 注意

- 作戦圏は、実際に移動できる範囲とは完全一致しない。
- 作戦圏は軍師能力の候補計算に使う。

# 13. 描画方針

## 13.1 表示

- 盤面は isometric 風に表示する。
- ただし、ロジックは必ず2Dグリッドで管理する。
- `gridToIso(x,y)` で描画座標へ変換する。

## 13.2 素材方針

初期実装では画像素材を大量生成しない。

- 地上マス: SVG/CSSの菱形タイル
- 湖マス: SVG/CSSの菱形タイル
- 拠点: 2×2の簡易台座
- 橋: SVG/CSSの板状パーツ
- 障害物: 簡易アイコン
- 駒: 共通トークン + チーム色 + 兵種文字

各チーム×兵種ごとの画像素材は作らない。  
まずは共通コンポーネントで描画する。

---

# 14. Codex向け禁止事項

- isometric表示座標をゲームロジックに使わない。
- 拠点内の駒を通常座標に置かない。
- 橋を任意座標へ自由生成しない。
- BridgeSlotなしで橋を生成しない。
- `P` を拠点として扱わない。
- `D` を特殊地形として扱わない。移動上は通常の道である。
- 湖 `~` を通常駒が移動可能にしない。
- 中立CPU拠点に生産や移動をさせない。
- 各チーム×兵種の画像素材が必須であるかのように実装しない。
