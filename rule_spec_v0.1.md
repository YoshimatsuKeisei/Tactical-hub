# ルール仕様書 v0.1

この文書は、戦略陣取りゲームの実装前ルール仕様書です。  
Codex などのコード生成エージェントに渡す「ルール正本」として使うことを想定しています。

本仕様書では、これまでの会話で確定・準確定したルールを整理します。  
未確定の項目は `TBD` または `暫定` と明記します。

# 0. 実装方針

## 0.1 最重要方針

- 本作は、3〜5人を想定した同時行動型の戦略陣取りゲームである。
- 各プレイヤーが順番に行動する交互ターン制ではなく、各フェーズで全プレイヤーが入力し、その後にまとめて解決する。
- ゲームロジックは常に2Dグリッド座標で管理する。
- isometric 表示は描画専用であり、移動判定・射程判定・攻撃判定・占有判定に画面座標を使ってはならない。
- UIや演出よりも、まずルール処理の正確性を優先する。
- 仕様にない兵種・効果・勝利条件を勝手に追加しない。

## 0.2 用語

| 用語       | 意味                                           |
| ---------- | ---------------------------------------------- |
| 駒         | 盤面上のユニット                               |
| 拠点       | 2×2の拠点エリア。内部リストで駒を管理する      |
| 本拠地     | 各チームの初期拠点。奥座敷枠を持つ             |
| 中継拠点   | 初期状態では中立CPUが守る拠点                  |
| 道マス     | 通常の地上移動が可能なマス                     |
| 湖/水面    | 通常駒は移動不可。忍者のみ水面移動可能         |
| 橋         | 建設軍師が湖上に設置する通行可能マス           |
| 障害物     | 建設軍師が設置する移動・近接攻撃阻害物         |
| 作戦圏     | 自軍拠点からつながる道、およびその道に面する湖 |
| 水計       | 橋リセットにより橋上駒を水没させる戦術         |
| 占拠優先権 | 空になった拠点を占拠できるチームの権利         |

# 1. ゲーム概要

## 1.1 対応人数

- 対応人数は3〜5人。
- まずは4人用マップを中心に調整する想定。
- 3人/5人マップは後続展開とする。

## 1.2 基本構造

1ターンは複数フェーズに分かれる。

1. 生産フェーズ
2. 移動入力フェーズ
3. 移動解決フェーズ
4. 攻撃対象決定フェーズ
5. 戦闘解決フェーズ
6. 軍師アクション処理フェーズ
7. 後処理フェーズ
8. 試合終了判定

## 1.3 試合形式

試合形式は以下の2つを想定する。

| 形式         | 内容                                     | 状態     |
| ------------ | ---------------------------------------- | -------- |
| 制限ターン制 | 一定ターン終了時に得点で順位を決める     | 採用予定 |
| 完全決着制   | 最後まで生き残る、または王撃破で決着する | 採用予定 |

標準ルールは制限ターン制を想定する。  
完全決着制はフレンド戦・上級者向け・CPU戦などで使う想定。

# 2. プレイヤー/チーム

## 2.1 チーム

- 各プレイヤーは1チームを操作する。
- 各チームにはチームカラーがある。
- 駒本体はチームカラーで表示する。
- 兵種は駒に表示された文字で識別する。

## 2.2 脱落

- 王が撃破されたチームは脱落する。
- 脱落後、プレイヤーは以下を選べる。
  - 観戦する
  - 離脱する
- 離脱した場合でも、試合終了後に結果通知を確認できる。

## 2.3 観戦

- 観戦者には公開情報を表示する。
- 未解決の予約行動や非公開情報は、通常オンライン対戦では表示しない方針とする。

# 3. マップ/地形

## 3.1 グリッド

- 盤面は2Dグリッドで管理する。
- 斜め移動は可能であり、斜め1マスも1マス移動として扱う。
- 距離計算は原則としてチェビシェフ距離を使う。

```text
距離 = max(abs(dx), abs(dy))
```

## 3.2 地形種別

| 地形    | 移動                   | 備考                 |
| ------- | ---------------------- | -------------------- |
| 道マス  | 通常移動可能           | 基本地形             |
| 湖/水面 | 通常駒は不可           | 忍者のみ水面移動可能 |
| 橋      | 通常移動可能           | 敵味方問わず通行可能 |
| 拠点    | 拠点内リストとして管理 | 通常マスとは別扱い   |
| 障害物  | 移動不可               | 遠距離攻撃は通る     |

## 3.3 湖

- 湖は通常駒では移動できない。
- 忍者のみ水面上を移動できる。
- 空いているマップ領域は湖として扱う方針。
- 湖は広くつながっていてもよい。
- 水面上の忍者は攻撃不可・兵種入れ替え不可・拠点占拠不可。

## 3.4 作戦圏

作戦圏は軍師能力の参照範囲である。

### 作戦圏の定義

- 自軍拠点からつながる道マス。
- その道マスに面している湖。
- 障害物は作戦圏のつながりを遮断しない。
- 敵拠点/中立拠点は作戦圏をまたいで通過できない。
- ただし、敵拠点を挟んだ先にも別の自軍拠点がある場合、その先の道は別起点の作戦圏として扱う。

### 注意

- 作戦圏は「実際に通行可能な範囲」と完全一致しない。
- 軍師の橋・障害物・転送の計算に使うための概念である。

# 4. 拠点

## 4.1 拠点の基本

- 拠点は見た目上2×2マス。
- ただし、戦闘・保持管理では「内部リスト」として扱う。
- 通常拠点は最大4体まで駒を保持できる。
- 拠点内の駒は、拠点内で個別座標を持たない。

## 4.2 拠点内リスト

- 拠点内の駒は `baseId` と `slotId` で管理する。
- 拠点内の敵を攻撃する場合、拠点内リストから攻撃対象を選ぶ。
- 拠点内の空き枠がある場合のみ、生産や入城が可能。

## 4.3 拠点攻撃判定

- 拠点は攻撃判定上「大きな1マス」として扱う。
- 駒から拠点までの距離は、拠点を構成する4マスのうち最も近いマスとのチェビシェフ距離で判定する。
- 拠点に接している場合、射程1として扱う。
- 射程内に拠点が入っていれば、拠点内の敵駒を攻撃対象にできる。
- 拠点内のどの駒を攻撃するかは、攻撃対象選択で決める。

## 4.4 拠点による遮蔽

- 拠点を挟んで反対側にいる外部駒は攻撃できない。
- 距離上は射程内であっても、拠点の向こう側の外部駒は攻撃対象にできない。
- 工兵隊も、拠点を越えて外部駒を攻撃することはできない。
- 工兵隊は拠点内の敵を攻撃する兵種である。

## 4.5 本拠地奥座敷

本拠地には奥の1枠に相当する保護枠を持たせる。

### 奥座敷ルール

- 奥座敷にいる駒は、本拠地内に他の味方駒がいる限り、敵の攻撃対象に選ばれない。
- 本拠地内に奥座敷の駒しか残っていない場合、その駒は攻撃対象になる。
- 奥座敷の駒が攻撃しても、保護状態は解除されない。
- 王を置くことを想定するが、弓兵などを置くこともできる。

## 4.6 占拠優先権

- 空になった拠点は、占拠優先権を持つチームのみが占拠できる。
- 占拠は移動フェーズで行う。
- 占拠優先権を持たないチームは、基本その拠点を占拠できないが近くにいない場合は代わりに占拠することが可能。
- 敵拠点へ移動できるのは、敵駒が1体もいない場合に限る。

## 4.7 生産

- 駒は自軍拠点内の空き枠に生産される。
- 拠点内に空き枠があれば、完全包囲されていても生産可能。
- 占拠した拠点も生産場所として使える。
- ただし、拠点内が満員の場合は生産不可。

## 4.8 中立CPU拠点

中継拠点は第三者CPUチームが保有する。

### 中立守備隊

- 中立拠点には以下の3体を初期配置する。
  - 歩兵
  - 騎兵
  - 弓兵
- 中立守備隊は生産しない。
- 中立守備隊は移動しない。
- 中立守備隊は拠点外へ出ない。
- 射程内に入った敵を攻撃する。
- 中立守備隊が全滅すると、その拠点は空拠点となり、占拠優先権が発生する。

# 5. 駒一覧

## 5.1 兵種表

| 兵種       | 表示文字  | 体力 | 移動 | 射程 | 生産上限 | 主な役割                 |
| ---------- | --------- | ---: | ---: | ---: | -------- | ------------------------ |
| 王         | 王        |    3 |    1 |    1 | 1        | 敗北条件                 |
| 歩兵       | 歩        |    1 |    1 |    1 | なし     | 基本戦力/撤退            |
| 騎兵       | 馬        |    1 | 1〜2 |    1 | なし     | 機動力                   |
| 弓兵       | 弓        |    1 |    1 |    3 | 暫定3    | 遠距離/対騎兵            |
| 工兵隊     | 工        |    1 |    1 |    5 | 暫定2    | 対拠点                   |
| 忍者       | 忍        |    1 |    1 |    1 | 1        | 水面移動/兵種入れ替え    |
| 忍者見習い | 見 / 忍見 |    1 |    1 |    1 | なし     | 忍者入れ替えによる弱体駒 |
| 軍師       | 帥        |    1 |    1 |    - | 暫定1    | 支援/建設/転送           |

## 5.2 王

- 王は各チーム1体。
- 王の体力は3。
- ダメージを受けるたびに体力が1減る。
- 体力が0になると撃破される。
- 王が撃破されたチームは敗北/脱落する。
- 王が橋上で水計に巻き込まれた場合も撃破扱いとなる。

## 5.3 歩兵

- 体力1。
- 移動1。
- 射程1。
- 斜めも射程1に含む。
- 弓兵に有利。
- 忍者に有利。
- 撤退権を持つ。

### 歩兵の撤退適性

- 歩兵は撤退状態になれる。
- 撤退中の歩兵は攻撃できない。
- 歩兵は撤退中、防御補正を得る。
- 詳細は「撤退ルール」を参照。

## 5.4 騎兵

- 体力1。
- 移動は1〜2マスを選択可能。
- 射程1。
- 斜め移動は1マス扱い。
- 2マス移動時、1マス目に障害物や通行不可マスがある場合、その先には移動できない。
- ジャンプ移動は不可。
- 歩兵に有利。
- 忍者に有利。
- 弓兵に不利。
- 生産上限は設けない。

## 5.5 弓兵

- 体力1。
- 移動1。
- 射程3。
- 騎兵に有利。
- 忍者に有利。
- 遠距離攻撃を行う。
- 障害物を越えて攻撃できる。
- 拠点を挟んで向こう側の外部駒は攻撃できない。

## 5.6 工兵隊

- 体力1。
- 移動1。
- 射程5。
- 城攻め/拠点攻撃に特化する。
- 拠点内の敵駒を攻撃対象にできる。
- 通常の地上戦では攻撃できない。
- 拠点内の王は、暫定的に工兵隊の攻撃対象外とする。
- 橋上や地上で敵に狙われた場合、防御能力は低い。

## 5.7 忍者

- 体力1。
- 移動1。
- 射程1。
- 水面上を移動できる。
- 水面移動中は、非忍者から攻撃されない。
- 水面上では攻撃不可。
- 水面上では兵種入れ替え不可。
- 水面上では拠点占拠不可。
- 忍者同士は水面上で攻撃可能。
- 水面から上陸した後は、通常地上の駒として攻撃可能。
- 橋を水面として横切ることはできない。
- 水面上の忍者は、橋から上陸できない。

### 忍者の兵種入れ替え

- 忍者は、3マス以内の敵駒と兵種を入れ替えられる。
- 対象は歩兵・騎兵・弓兵のみ。
- 王・軍師・工兵隊は対象外。
- 合成駒は暫定的に対象外とする。
- 位置は入れ替えない。
- 入れ替えるのは兵種のみ。
- 自軍忍者が敵騎兵を対象にした場合、自軍忍者は騎兵となり、敵騎兵は忍者見習いとなる。

## 5.8 忍者見習い

- 忍者の兵種入れ替えによって発生する。
- 通常忍者とは別兵種として扱う。
- 忍者の1体上限には含まれない。
- 生産不可。
- 水面移動不可。
- 兵種入れ替え不可。
- 通常地上でのみ行動する弱体化駒。

## 5.9 軍師

軍師は攻撃権を持たない支援駒である。

- 体力1。
- 移動1。
- 射程なし。
- 攻撃不可。
- 攻撃対象にされた場合、一方的に撃破され得る。
- 生産時に役割を選択する。

### 軍師の役割

軍師は以下のいずれかの役割を持つ。

| 役割     | 内容                                     |
| -------- | ---------------------------------------- |
| 鼓舞軍師 | 周囲2マス以内の味方を支援する            |
| 建設軍師 | 橋・障害物を設置/リセットする            |
| 転送軍師 | 条件を満たす味方駒を作戦圏内で移動させる |

### 鼓舞軍師

- 周囲2マス以内の味方駒に鼓舞効果を与える。
- 鼓舞中の味方は移動優先順が上がる。
- 鼓舞中の味方は命中率が1/6から1/5になる。
- 命中率上昇は重複しない。
- 複数鼓舞による移動優先順上昇は、上級ルールでのみ検討する。

### 建設軍師

- 橋を1つ維持できる。
- 障害物を1つ維持できる。
- 橋と障害物のクールダウンは別管理。
- 橋/障害物の設置・リセットは、軍師アクション処理フェーズで行う。
- 1ターンに実行できる建設アクションは暫定的に1つまで。
- 建設軍師が撃破された場合、その軍師が管理していた橋・障害物は戦闘後処理で消滅する。

### 転送軍師

TBD。暫定仕様は以下。

- 自軍の歩兵・騎兵・弓兵・忍者を対象にできる。
- 王・軍師・工兵隊は対象外。
- 作戦圏内で、移動可能な場所を自動計算し、選択可能にする。
- 拠点間の移動も可能。
- 対象や移動先の詳細は、今後のテストで調整する。

# 6. 生産ルール

## 6.1 生産タイミング

- 生産は定期的に行う。
- 生産間隔は暫定で5ターンに1回とする。
- 調整範囲は4〜6ターン。
- 生産速度が速すぎると拠点攻めが成立しにくいため、初期値は5ターンを推奨する。

## 6.2 生産場所

- 生産は自軍拠点内の空き枠に行う。
- 自軍拠点に空き枠があれば、敵に包囲されていても生産可能。
- 占拠した拠点も生産場所として使える。
- ただし、拠点内が満員の場合は生産不可。

## 6.3 生産兵種

- 生産兵種は自由選択。
- ただし、生産上限に達している兵種は選択不可。
- 王は追加生産不可。
- 忍者見習いは生産不可。

# 7. 移動ルール

## 7.1 移動入力

- 各駒をタップ/クリックすると、その駒の移動候補が自動表示される。
- 移動しない選択も可能。
- 動かしたい駒だけ動かす。
- 1ターンに動かす駒数の制限は設けない。

## 7.2 移動候補条件

移動先候補は以下をすべて満たす必要がある。

1. 敵兵・自兵がいない。
2. 通行可能なマスである。
3. 湖は忍者以外不可。
4. 橋は通常通行可能。
5. 軍師が設置した障害物がない。
6. 敵拠点へ入る場合、その拠点内に敵駒が1体もいない。
7. 敵拠点へ入る場合、占拠優先権などの条件を満たしている。

## 7.3 現在位置の扱い

- 移動候補計算では、移動予定の有無に関係なく、現在駒が存在するマスは占有済みとして扱う。
- そのため通常移動による位置交換は発生しない。
- 「相手がどく予定だから、その場所へ入る」という処理は行わない。

## 7.4 移動解決

- 移動解決時点で、移動先が合法か再判定する。
- 移動優先順の高い駒から順に処理する。
- 解決時点で移動先が埋まっている場合、その駒は移動失敗し、現在位置に残る。
- 同じ移動優先度の駒が衝突した場合、ランダムで優先側を決める。

## 7.5 移動優先順

基本移動優先順は以下。

```text
王 > 歩兵 > 騎兵 > 弓兵 > 忍者 > 工兵隊 > 軍師
```

鼓舞軍師の支援を受けている駒は、移動優先順が上がる。  
詳細な優先順の計算は今後の実装時に調整する。

# 8. 攻撃/戦闘ルール

## 8.1 攻撃対象決定

- 攻撃対象は移動解決後の盤面をもとに決定する。
- 射程内に敵が1体しかいない場合、自動選択される。
- 射程内に敵が複数いる場合、自動優先で対象が選ばれる。
- プレイヤーは必要に応じて攻撃対象を変更できる。
- 攻撃対象の自動優先は、王・有利対象・危険駒・HP低下駒などを基準にする。詳細はTBD。

## 8.2 射程

- 射程はチェビシェフ距離で判定する。
- 射程1は斜めを含む周囲8マス。
- 弓兵は射程3。
- 工兵隊は射程5。
- 拠点攻撃時は、拠点全体を大きな1マスとして扱う。

## 8.3 攻撃有利関係

| 攻撃側 | 有利対象           |
| ------ | ------------------ |
| 王     | 全兵種             |
| 歩兵   | 弓兵、忍者         |
| 騎兵   | 歩兵、忍者         |
| 弓兵   | 騎兵、忍者         |
| 工兵隊 | 拠点内の王以外の駒 |
| 忍者   | 軍師、工兵隊       |
| 軍師   | 攻撃不可           |

## 8.4 命中率

- 基本命中率は1/6。
- 鼓舞軍師の支援を受けている場合、命中率は1/5。
- 命中率上昇は重複しない。
- 1回の命中で体力を1減らす。
- 体力1の駒は命中すると撃破される。
- 王は体力3のため、3回命中すると撃破される。

### 撤退中歩兵への攻撃

- 攻撃対象が撤退中の歩兵である場合、攻撃側の最終命中率を半減する。
- 通常攻撃は1/6から1/12になる。
- 鼓舞攻撃は1/5から1/10になる。
- この補正は歩兵の撤退中にのみ適用する。
- 撤退中の歩兵は攻撃できない。

## 8.5 複数攻撃

- 1体の敵に対し、複数の味方駒が攻撃対象を指定できる。
- 拠点攻防戦では集中攻撃が重要な戦術となる。
- 複数攻撃により、低い命中率でも撃破確率を高められる。

## 8.6 戦闘解決方針

戦闘解決は以下の方針で行う。

- 攻撃イベントを作成する。
- 攻撃イベントには優先階層を与える。
- 上位階層の攻撃は下位階層より先に解決する。
- 同階層の攻撃は同時解決する。
- 先に撃破された駒の未解決攻撃はキャンセルされる。
- ただし、同階層内で撃破された場合、その同階層攻撃は発生する。

### 暫定攻撃階層

| 階層 | 内容                         |
| ---- | ---------------------------- |
| S    | 工兵隊の拠点攻撃など特殊攻撃 |
| A    | 有利相性の攻撃、王の攻撃     |
| B    | 通常攻撃                     |
| C    | 不利側/低優先攻撃            |

この階層は実装テストで調整する。

# ※撤退ルール（暫定）

## 撤退状態の位置づけ

撤退状態は、歩兵専用ではなく全兵種に発生し得る共通状態である。  
ただし、撤退中に防御補正を得るのは歩兵のみである。

このルールは暫定仕様であり、実際のプレイテストで有効性を確認してから最終確定する。

## 撤退状態の共通効果

撤退中の駒には、兵種を問わず以下を適用する。

- 撤退中の駒は攻撃できない。
- 撤退中の駒は攻撃対象になり得る。
- 撤退中の駒が自軍拠点へ到達した場合、拠点内へ入り、撤退状態は解除される。
- 撤退中の駒が敵拠点側へ移動した場合、撤退状態は自動解除される。
- 攻撃対象決定フェーズで撤退中の駒が敵と接している場合、プレイヤーは撤退を維持するか、撤退を解除して攻撃するかを選択できる。

## 撤退状態の発動条件

駒は、以下をすべて満たす場合に撤退中になる。

1. この駒が直前の戦闘で敵拠点から3マス以内の戦闘に参加している。
2. 次の移動入力で、この駒が自軍拠点方向へ移動する。
3. 自軍拠点方向への移動とは、移動後の位置が移動前よりも撤退先自軍拠点に近づくことを意味する。

## 歩兵の撤退補正

歩兵は撤退に適した兵種であり、撤退中に防御補正を得る。

- 通常命中率1/6の攻撃は、撤退中の歩兵に対しては1/12になる。
- 攻撃側に鼓舞などの命中率上昇がある場合、最終命中率を半減する。
  - 例：1/5 → 1/10
- この防御補正は歩兵にのみ適用される。

## 歩兵以外の撤退

歩兵以外の駒も撤退状態になり得る。  
ただし、歩兵以外の駒は撤退中に防御補正を得ない。

- 騎兵、弓兵、忍者、工兵隊、軍師は撤退中に攻撃できない。
- これらの駒は撤退中でも通常の命中率で攻撃を受ける。
- つまり、歩兵以外の撤退は「防御特化」ではなく、「攻撃できない退避状態」である。

# 9. 橋ルール

## 9.1 橋の設置

- 橋は建設軍師が設置する。
- 橋はマップ上で指定された同一湖内の直線区間にのみ設置できる。
- 橋は縦または横一直線のみ。
- 斜め橋は不可。
- 橋の長さは数値上限ではなく、マップ側の橋候補スロットで制御する。

## 9.2 橋の設置範囲

橋は以下の条件を満たす場所に設置できる。

1. 始点側が自軍作戦圏の道マスに接する湖である。
2. 終点側が同じ湖に接する別の道である。
3. 終点側の道が自軍作戦圏である必要はない。
4. 橋はマップ定義済みの橋候補スロットにのみ設置できる。

## 9.3 橋の通行

- 橋は敵味方を問わず通行可能。
- 橋上の駒は攻撃可能。
- 橋上の駒は攻撃対象になる。
- 橋は通常地上マスに近い扱いだが、水計対象となる特殊地形である。

## 9.4 忍者と橋

- 忍者は水面移動中、橋を水面として横切れない。
- 忍者は水面から橋へ上陸できない。
- 忍者が地上側から橋へ通常移動する扱いについては、実装時に明確化する。

## 9.5 橋リセット

- 橋を別の場所へ移動させることは即時にはできない。
- 別の場所に橋を作りたい場合、まず現在の橋をリセット消去する。
- 橋リセット後、橋は5ターンのクールダウンに入る。
- クールダウン終了後、橋は未設置状態となり、新たに設置可能になる。
- 5ターンは初期値であり、調整可能。

## 9.6 橋リセット不可条件

以下の場合、橋リセットは選択不可とする。

- 自軍王が現在の橋上にいる。
- その橋を管理する建設軍師本人が現在の橋上にいる。
- 橋が存在しない。
- 橋がすでにクールダウン中である。

UIでは選択不可としてグレー化する。

## 9.7 水計

- 橋リセット時、橋上にいる駒は敵味方を問わず水没し除去される。
- 橋上の敵駒が水没した場合、橋所有者に撃破点が入る。
- 橋上の味方駒が水没した場合、敵にも自軍にも撃破点は入らない。
- 敵王が橋上で水没した場合、王撃破扱いとなる。
- 自軍王や管理軍師本人が橋上にいる場合、そもそも橋リセットできない。

## 9.8 橋の消滅

以下の場合、橋は消滅する。

- 橋を管理する建設軍師が撃破される。
- そのチームの王が撃破される。
- 橋がリセットされる。

軍師死亡/王死亡による橋消滅は、戦闘解決後の後処理で行う。

# 10. 障害物ルール

## 10.1 障害物の設置

- 障害物は建設軍師が設置する。
- 障害物は1チームにつき1つまで維持できる。
- 障害物は道マスまたは橋上に設置できる。
- 敵の橋上にも障害物を設置できる。
- 水面そのものには障害物を設置できない。
- 拠点内には障害物を設置できない。

## 10.2 設置条件

障害物を設置できるマスは以下を満たす。

1. 敵兵・自兵がいない。
2. 敵拠点・自拠点ではない。
3. 水面上ではない。
4. 既存の障害物がない。
5. 設置可能な道マスまたは橋上である。

## 10.3 障害物の効果

- 障害物は移動を妨げる。
- 障害物は近接攻撃を妨げる。
- 弓兵・工兵隊などの遠距離攻撃は障害物を越えて行える。
- 障害物は自軍にも敵軍にも影響する。

## 10.4 障害物リセット

- 障害物を別の場所へ動かす場合、まず現在の障害物をリセット消去する。
- 障害物リセット後、5ターンのクールダウンに入る。
- 橋のクールダウンと障害物のクールダウンは別管理。

## 10.5 橋上障害物

- 橋上に障害物を置ける。
- 障害物をリセットしても橋は残る。
- 橋が消滅またはリセットされた場合、その橋上の障害物も消滅する。
- 橋上の障害物が敵のものであっても、橋リセット時にまとめて消滅する。

# 11. 軍師アクション処理

## 11.1 処理タイミング

軍師アクションは、他の駒の戦闘解決後に処理する。

処理順は以下。

1. 戦闘解決
2. 軍師アクション処理
3. 軍師死亡/王死亡による橋・障害物消滅
4. 水計/消滅/得点処理
5. 後処理
6. 次ターンへ

## 11.2 建設アクション

建設軍師は暫定的に1ターンに1つの建設アクションを行える。

対象アクション例：

- 橋を設置する
- 橋をリセットする
- 障害物を設置する
- 障害物をリセットする

# 15. UI/入力に関わるルール

## 15.1 移動候補表示

- 駒を選択すると移動候補が表示される。
- 移動候補はロジック上の2D座標から計算する。
- isometric表示上の座標から移動候補を計算してはならない。

## 15.2 攻撃候補表示

- 攻撃対象は移動解決後の盤面をもとに表示する。
- 拠点攻撃時は、拠点内ユニット一覧を表示し、対象を選べるようにする。
- 奥座敷で保護中の駒は、攻撃対象一覧から除外する。

### 撤退中歩兵の攻撃確認

攻撃対象決定フェーズで撤退中の歩兵を選択し、その歩兵が敵と接している場合、以下の確認UIを表示する。

「この歩兵は撤退中です。撤退を解除して攻撃しますか？」

選択肢:

- 防御継続
  - 撤退中を維持する。
  - この歩兵は攻撃しない。
  - 被命中率低下は維持される。
- 撤退解除して攻撃
  - 撤退中を解除する。
  - この歩兵は通常状態として攻撃可能になる。
  - 被命中率は通常に戻る。

## 15.3 タップ/クリック判定

- 見た目レイヤーと当たり判定レイヤーを分ける。
- 駒・橋・障害物・拠点・タイルの判定優先順位を固定する。
- ハイライトと当たり判定は同じ gridToIso 変換を使う。

推奨優先順位:

1. UIボタン
2. 駒
3. 障害物
4. 橋
5. 拠点
6. 移動候補タイル
7. 通常タイル
8. 湖

# 17. Codex向け禁止事項

- 交互ターン制に変更しない。
- 拠点内駒を通常の2D座標駒として扱わない。
- 本拠地奥座敷の保護を「攻撃したら解除されるシールド」と解釈しない。
- 橋リセット後に即再設置できるようにしない。
- 橋/障害物のクールダウンを共通管理にしない。
- 忍者が水面上から攻撃・兵種入れ替え・拠点占拠できるようにしない。
- 工兵隊が通常地上戦で攻撃できるようにしない。
- isometric表示の画面座標をゲームロジックに使わない。
- 仕様にない兵種・効果・勝利条件を追加しない。
- ユーザーが指定していないUI/ルールを勝手に補完しない。
