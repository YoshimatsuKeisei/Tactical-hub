# ルール仕様書 v0.1

この文書は、戦略陣取りゲームの実装前ルール仕様書です。  
Codex などのコード生成エージェントに渡す「ルール正本」として使うことを想定しています。

本仕様書では、これまでの会話で確定・準確定したルールを整理します。  
未確定の項目は `TBD` または `暫定` と明記します。

## Phase 3-A: 撤退状態・撤退開始・撤退解除

Phase 3-A では、撤退を全兵種に発生し得る共通状態として扱う。
撤退中の駒は攻撃できず、AttackIntent が残っていても BattleEvent には変換しない。
撤退中の駒は通常どおり攻撃対象になり得る。

撤退資格は戦闘解決後に、機械判定用のターンフラグとして保存する。
対象になるのは、その戦闘開始時点で生存し、有効な BattleEvent の攻撃者または対象になり、戦闘後も生存しており、敵拠点から3マス以内の通常地上または橋上にいる駒である。
水面上、拠点内、removed 状態の駒は撤退資格を得ない。
敵拠点との距離、自軍拠点との距離はいずれもチェビシェフ距離で判定する。

撤退開始は次の移動解決時に行う。
撤退資格を持つ駒が、最寄りの自軍支配拠点へ近づく合法移動を行った場合、`retreating` 状態になる。
最寄り自軍拠点は本拠地と占領済み中継拠点を同等に扱う。
移動が不成立だった場合、撤退状態は新規付与しない。

撤退中の移動では、最寄り自軍支配拠点への距離が同じか短くなる合法移動では撤退を維持する。
距離が長くなる合法移動を行った場合、撤退状態は解除される。
自軍支配拠点へ入った場合、拠点内へ入り撤退状態は解除される。
移動入力で明示的に stay を選んだ場合も撤退状態を解除する。

Phase 3-A では、撤退中の駒に対する防御確率補正は実装しない。
攻撃成功確率表、鼓舞補正、完全同時戦闘方式は変更しない。

撤退資格の交戦参加判定は、攻撃の命中/失敗に依存しない。
有効な BattleEvent の攻撃者と対象者は、命中しなかった場合でも交戦参加者として扱う。
敵支配拠点から3マス以内かどうかは、BattleEvent 確定時点の位置で判定し、次の移動解決時に再判定しない。
敵支配拠点は、敵チームの本拠地と、敵チームの `controlledBaseIds` に含まれる占領済み中継拠点を含む。

# 0. 実装方針

## 0.1 最重要方針

- 本作は、3〜5人を想定した同時行動型の戦略陣取りゲームである。
- 各プレイヤーが順番に行動する交互ターン制ではなく、各フェーズで全プレイヤーが入力し、その後にまとめて解決する。
- ゲームロジックは常に2Dグリッド座標で管理する。
- isometric 表示は描画専用であり、移動判定・射程判定・攻撃判定・占有判定に画面座標を使ってはならない。
- UIや演出よりも、まずルール処理の正確性を優先する。
- 仕様にない兵種・効果・勝利条件を勝手に追加しない。

## 0.2 用語

| 用語       | 意味                                           |
| ---------- | ---------------------------------------------- |
| 駒         | 盤面上のユニット                               |
| 拠点       | 2×2の拠点エリア。内部リストで駒を管理する      |
| 本拠地     | 各チームの初期拠点。奥座敷枠を持つ             |
| 中継拠点   | 初期状態では中立CPUが守る拠点                  |
| 道マス     | 通常の地上移動が可能なマス                     |
| 湖/水面    | 通常駒は移動不可。忍者のみ水面移動可能         |
| 橋         | 建設軍師が湖上に設置する通行可能マス           |
| 障害物     | 建設軍師が設置する移動・近接攻撃阻害物         |
| 作戦圏     | 自軍拠点からつながる道、その道に面する湖、およびその道へ接続するactiveな橋 |
| 水計       | 橋リセットにより橋上駒を水没させる戦術         |
| 占拠優先権 | 空になった拠点を占拠できるチームの権利         |

# 1. ゲーム概要

## 1.1 対応人数

- 対応人数は3〜5人。
- まずは4人用マップを中心に調整する想定。
- 3人/5人マップは後続展開とする。

## 1.2 基本構造

1ターンは複数フェーズに分かれる。

1. 生産フェーズ
2. 移動入力フェーズ
3. 移動解決フェーズ
4. 攻撃対象決定フェーズ
5. 戦闘解決フェーズ
6. 軍師アクション処理フェーズ
7. 後処理フェーズ
8. 試合終了判定

## 1.3 試合形式

試合形式は以下の2つを想定する。

| 形式         | 内容                                     | 状態     |
| ------------ | ---------------------------------------- | -------- |
| 制限ターン制 | 一定ターン終了時に得点で順位を決める     | 採用予定 |
| 完全決着制   | 最後まで生き残る、または王撃破で決着する | 採用予定 |

標準ルールは制限ターン制を想定する。  
完全決着制はフレンド戦・上級者向け・CPU戦などで使う想定。

# 2. プレイヤー/チーム

## 2.1 チーム

- 各プレイヤーは1チームを操作する。
- 各チームにはチームカラーがある。
- 駒本体はチームカラーで表示する。
- 兵種は駒に表示された文字で識別する。

## 2.2 脱落

- 王が撃破されたチームは脱落する。
- 脱落後、プレイヤーは以下を選べる。
  - 観戦する
  - 離脱する
- 離脱した場合でも、試合終了後に結果通知を確認できる。

## 2.3 観戦

- 観戦者には公開情報を表示する。
- 未解決の予約行動や非公開情報は、通常オンライン対戦では表示しない方針とする。

# 3. マップ/地形

## 3.1 グリッド

- 盤面は2Dグリッドで管理する。
- 斜め移動は可能であり、斜め1マスも1マス移動として扱う。
- 距離計算は原則としてチェビシェフ距離を使う。

```text
距離 = max(abs(dx), abs(dy))
```

## 3.2 地形種別

| 地形    | 移動                   | 備考                 |
| ------- | ---------------------- | -------------------- |
| 道マス  | 通常移動可能           | 基本地形             |
| 湖/水面 | 通常駒は不可           | 忍者のみ水面移動可能 |
| 橋      | 通常移動可能           | 敵味方問わず通行可能 |
| 拠点    | 拠点内リストとして管理 | 通常マスとは別扱い   |
| 障害物  | 移動不可               | 遠距離攻撃は通る     |

## 3.3 湖

- 湖は通常駒では移動できない。
- 忍者のみ水面上を移動できる。
- 空いているマップ領域は湖として扱う方針。
- 湖は広くつながっていてもよい。
- 水面上の忍者は攻撃不可・兵種入れ替え不可・拠点占拠不可。

## 3.4 作戦圏

作戦圏は軍師能力の参照範囲である。

### 作戦圏の定義

- 自軍拠点からつながる道マス。
- その道マスに面している湖。
- その道マスへ接続しているactiveな橋タイル。橋の所有チームは問わない。
- 障害物は作戦圏のつながりを遮断しない。
- 敵拠点/中立拠点は作戦圏をまたいで通過できない。
- ただし、敵拠点を挟んだ先にも別の自軍拠点がある場合、その先の道は別起点の作戦圏として扱う。
- 橋タイル自体を作戦圏へ含めても、橋の対岸にある別道路区間までは自動的に作戦圏へ追加しない。

### 注意

- 作戦圏は「実際に通行可能な範囲」と完全一致しない。
- 軍師の橋・障害物・転送の計算に使うための概念である。

# 4. 拠点

## 4.1 拠点の基本

- 拠点は見た目上2×2マス。
- ただし、戦闘・保持管理では「内部リスト」として扱う。
- 通常拠点は最大4体まで駒を保持できる。
- 拠点内の駒は、拠点内で個別座標を持たない。

## 4.2 拠点内リスト

- 拠点内の駒は `baseId` と `slotId` で管理する。
- 拠点内の敵を攻撃する場合、拠点内リストから攻撃対象を選ぶ。
- 拠点内の空き枠がある場合のみ、生産や入城が可能。

## 4.3 拠点攻撃判定

- 拠点は攻撃判定上「大きな1マス」として扱う。
- 駒から拠点までの距離は、拠点を構成する4マスのうち最も近いマスとのチェビシェフ距離で判定する。
- 拠点に接している場合、射程1として扱う。
- 射程内に拠点が入っていれば、拠点内の敵駒を攻撃対象にできる。
- 拠点内のどの駒を攻撃するかは、攻撃対象選択で決める。

## 4.4 拠点による遮蔽

- 拠点を挟んで反対側にいる外部駒は攻撃できない。
- 距離上は射程内であっても、拠点の向こう側の外部駒は攻撃対象にできない。
- 工兵隊も、拠点を越えて外部駒を攻撃することはできない。
- 工兵隊は拠点内の敵を攻撃する兵種である。

## 4.5 本拠地奥座敷

本拠地には奥の1枠に相当する保護枠を持たせる。

### 奥座敷ルール

- 奥座敷にいる駒は、本拠地内に他の味方駒がいる限り、敵の攻撃対象に選ばれない。
- 本拠地内に奥座敷の駒しか残っていない場合、その駒は攻撃対象になる。
- 奥座敷の駒が攻撃しても、保護状態は解除されない。
- 王を置くことを想定するが、弓兵などを置くこともできる。

## 4.6 占拠優先権

- 空になった拠点は、占拠優先権を持つチームのみが占拠できる。
- 占拠は移動フェーズで行う。
- 占拠優先権を持たないチームは、基本その拠点を占拠できないが近くにいない場合は代わりに占拠することが可能。
- 敵拠点へ移動できるのは、敵駒が1体もいない場合に限る。

## 4.7 生産

- 駒は自軍拠点内の空き枠に生産される。
- 拠点内に空き枠があれば、完全包囲されていても生産可能。
- 占拠した拠点も生産場所として使える。
- ただし、拠点内が満員の場合は生産不可。

## 4.8 中立CPU拠点

中継拠点は第三者CPUチームが保有する。

### 中立守備隊

- 中立拠点には以下の3体を初期配置する。
  - 歩兵
  - 騎兵
  - 弓兵
- 中立守備隊は生産しない。
- 中立守備隊は移動しない。
- 中立守備隊は拠点外へ出ない。
- 射程内に入った敵を攻撃する。
- 中立守備隊が全滅すると、その拠点は空拠点となり、占拠優先権が発生する。

# 5. 駒一覧

## 5.1 兵種表

| 兵種       | 表示文字  | 体力 | 移動 | 射程 | 生産上限 | 主な役割                 |
| ---------- | --------- | ---: | ---: | ---: | -------- | ------------------------ |
| 王         | 王        |    3 |    1 |    1 | 1        | 敗北条件                 |
| 歩兵       | 歩        |    1 |    1 |    1 | なし     | 基本戦力/撤退            |
| 騎兵       | 馬        |    1 | 1〜2 |    1 | なし     | 機動力                   |
| 弓兵       | 弓        |    1 |    1 |    3 | 暫定3    | 遠距離/対騎兵            |
| 工兵隊     | 工        |    1 |    1 |    5 | 暫定2    | 対拠点                   |
| 忍者       | 忍        |    1 |    1 |    1 | 1        | 水面移動/兵種入れ替え    |
| 忍者見習い | 見 / 忍見 |    1 |    1 |    1 | なし     | 忍者入れ替えによる弱体駒 |
| 軍師       | 帥        |    1 |    1 |    - | 暫定1    | 支援/建設/転送           |

## 5.2 王

- 王は各チーム1体。
- 王の体力は3。
- ダメージを受けるたびに体力が1減る。
- 体力が0になると撃破される。
- 王が撃破されたチームは敗北/脱落する。
- 王が橋上で水計に巻き込まれた場合も撃破扱いとなる。

## 5.3 歩兵

- 体力1。
- 移動1。
- 射程1。
- 斜めも射程1に含む。
- 弓兵に有利。
- 忍者に有利。
- 撤退権を持つ。

### 歩兵の撤退適性

- 歩兵は撤退状態になれる。
- 撤退中の歩兵は攻撃できない。
- 歩兵は撤退中、防御補正を得る。
- 詳細は「撤退ルール」を参照。

## 5.4 騎兵

- 体力1。
- 移動は1〜2マスを選択可能。
- 射程1。
- 斜め移動は1マス扱い。
- 2マス移動時、1マス目に障害物や通行不可マスがある場合、その先には移動できない。
- ジャンプ移動は不可。
- 歩兵に有利。
- 忍者に有利。
- 弓兵に不利。
- 生産上限は設けない。

## 5.5 弓兵

- 体力1。
- 移動1。
- 射程3。
- 騎兵に有利。
- 忍者に有利。
- 遠距離攻撃を行う。
- 障害物を越えて攻撃できる。
- 拠点を挟んで向こう側の外部駒は攻撃できない。

## 5.6 工兵隊

- 体力1。
- 移動1。
- 射程5。
- 攻城戦に特化する。
- 攻城戦とは、射程内にある敵拠点のBaseSlot内ユニットを攻撃する場合を指す。
- 拠点内の敵駒を攻撃対象にできる。
- 通常の地上戦では攻撃できない。
- 拠点内の王も攻撃対象にできる。
- 橋上や地上で敵に狙われた場合、防御能力は低い。

## 5.7 忍者

- 体力1。
- 移動1。
- 射程1。
- 水面上を移動できる。
- 水面移動中は、非忍者から攻撃されない。
- 水面上では攻撃不可。
- 水面上では兵種入れ替え不可。
- 水面上では拠点占拠不可。
- 忍者同士は水面上で攻撃可能。
- 水面から上陸した後は、通常地上の駒として攻撃可能。
- 橋を水面として横切ることはできない。
- 水面上の忍者は、橋から上陸できない。
- 水計後に湖へ残った忍者の通常の湖内移動、および橋へ上がれない・橋を横切れない処理の実装は後続Phaseで扱う。

### 忍者の兵種入れ替え

- 忍者は、3マス以内の敵駒と兵種を入れ替えられる。
- 対象は歩兵・騎兵・弓兵のみ。
- 王・軍師・工兵隊は対象外。
- 合成駒は暫定的に対象外とする。
- 位置は入れ替えない。
- 入れ替えるのは兵種のみ。
- 自軍忍者が敵騎兵を対象にした場合、自軍忍者は騎兵となり、敵騎兵は忍者見習いとなる。

## 5.8 忍者見習い

- 忍者の兵種入れ替えによって発生する。
- 通常忍者とは別兵種として扱う。
- 忍者の1体上限には含まれない。
- 生産不可。
- 水面移動不可。
- 兵種入れ替え不可。
- 通常地上でのみ行動する弱体化駒。

## 5.9 軍師

軍師は攻撃権を持たない支援駒である。

- 体力1。
- 移動1。
- 射程なし。
- 攻撃不可。
- 攻撃対象にされた場合、一方的に撃破され得る。
- 生産時に役割を選択する。
- 各チームの初期配置軍師は鼓舞型とする。
- 初期配置では、各本拠地に王と鼓舞型軍師のみを配置する。
- 王は本拠地のprotected slotに配置し、鼓舞型軍師は王とは別のBaseSlotに配置する。

### 軍師の役割

軍師は以下のいずれかの役割を持つ。

| 役割     | 内容                                     |
| -------- | ---------------------------------------- |
| 鼓舞軍師 | 周囲2マス以内の味方を支援する            |
| 建設軍師 | 橋・障害物を設置/リセットする            |
| 転送軍師 | 条件を満たす味方駒を作戦圏内で移動させる |

### 鼓舞軍師

- 鼓舞型軍師が拠点内、またはいずれかの拠点から2マス以内にいる場合、鼓舞対象範囲は軍師の周囲1マス以内とする。
- 鼓舞型軍師がすべての拠点から3マス以上離れている場合、鼓舞対象範囲は軍師の周囲2マス以内とする。
- 距離はチェビシェフ距離で判定する。
- 拠点は大きな1マスとして扱い、拠点との距離は拠点を構成するセルのうち最も近いセルまでの距離で判定する。
- 鼓舞対象に含む地形は、通常道、拠点内、拠点接続部、再編成エリア、斜め道、橋上とする。
- 水面上の駒は鼓舞対象に含めない。
- 鼓舞対象は同じチームの駒のみとし、敵チーム・中立チーム・removed状態の駒は対象外とする。
- 拠点内にいる鼓舞型軍師と同じ拠点内の味方駒との距離は0として扱う。
- 鼓舞型軍師自身は、鼓舞対象一覧には含めない。
- 鼓舞型軍師自身が橋上にいる場合も、鼓舞効果は使用可能とする。
- 鼓舞補正は攻撃成功確率の分母を1減らす。
  - 1/5 → 1/4
  - 1/6 → 1/5
  - 1/7 → 1/6
- 鼓舞補正は重複しない。複数の鼓舞範囲に入っても補正は最大1回とする。
- 鼓舞対象は、移動解決後、攻撃対象確定後、戦闘解決開始前の盤面を基準に確定する。
- 戦闘解決では、戦闘開始直前に鼓舞対象をスナップショットし、各BattleEventの最終攻撃成功確率を確定する。
- 鼓舞型軍師が同じ戦闘内で撃破されても、戦闘開始前に確定した鼓舞補正はその戦闘中維持される。
- 鼓舞中の味方の移動優先順上昇は、今後のプレイテスト対象として保留する。

### 軍師上限・役割編成の保留事項

- 軍師のチーム内生産上限を現在の実装値から変更するかは未確定とする。
- 軍師の正式な総数上限は未確定とする。
- 鼓舞型、建設型、転送型をそれぞれ複数生産できるかは未確定とする。
- 鼓舞範囲が重なる場所への生産禁止・移動禁止は未確定とし、今後のプレイテスト対象とする。
- 同時移動によって鼓舞範囲が重なった場合の優先処理は未確定とする。
- 軍師の役割選択UIと、軍師2体以上の具体的な編成ルールは未確定とする。

### 建設軍師

- 橋を1つ維持できる。
- 障害物を1つ維持できる。
- 橋と障害物のクールダウンは別管理。
- 橋/障害物の設置・リセットは、軍師アクション処理フェーズで行う。
- 1ターンに実行できる建設アクションは暫定的に1つまで。
- 建設軍師が撃破された場合、その軍師が管理していた橋・障害物は戦闘後処理で消滅する。

# 6. 生産ルール

## 6.1 生産タイミング

- 生産は定期的に行う。
- 生産間隔は暫定で5ターンに1回とする。
- 調整範囲は4〜6ターン。
- 生産速度が速すぎると拠点攻めが成立しにくいため、初期値は5ターンを推奨する。

## 6.2 生産場所

- 生産は自軍拠点内の空き枠に行う。
- 自軍拠点に空き枠があれば、敵に包囲されていても生産可能。
- 占拠した拠点も生産場所として使える。
- ただし、拠点内が満員の場合は生産不可。

## 6.3 生産兵種

- 生産兵種は自由選択。
- ただし、生産上限に達している兵種は選択不可。
- 王は追加生産不可。
- 忍者見習いは生産不可。

# 7. 移動ルール

## 7.1 移動入力

- 各駒をタップ/クリックすると、その駒の移動候補が自動表示される。
- 移動しない選択も可能。
- 動かしたい駒だけ動かす。
- 1ターンに動かす駒数の制限は設けない。

## 7.2 移動候補条件

移動先候補は以下をすべて満たす必要がある。

1. 敵兵・自兵がいない。
2. 通行可能なマスである。
3. 湖は忍者以外不可。
4. 橋は通常通行可能。
5. 軍師が設置した障害物がない。
6. 敵拠点へ入る場合、その拠点内に敵駒が1体もいない。
7. 敵拠点へ入る場合、占拠優先権などの条件を満たしている。

## 7.3 現在位置の扱い

- 移動候補計算では、移動予定の有無に関係なく、現在駒が存在するマスは占有済みとして扱う。
- そのため通常移動による位置交換は発生しない。
- 「相手がどく予定だから、その場所へ入る」という処理は行わない。

## 7.4 移動解決

- 移動解決時点で、移動先が合法か再判定する。
- 移動優先順の高い駒から順に処理する。
- 解決時点で移動先が埋まっている場合、その駒は移動失敗し、現在位置に残る。
- 同じ移動優先度の駒が衝突した場合、ランダムで優先側を決める。

## 7.5 移動優先順

基本移動優先順は以下。

```text
王 > 歩兵 > 騎兵 > 弓兵 > 忍者 > 工兵隊 > 軍師
```

鼓舞軍師の支援を受けている駒は、移動優先順が上がる。  
詳細な優先順の計算は今後の実装時に調整する。

# 8. 攻撃/戦闘ルール

## 8.1 攻撃対象決定

- 攻撃対象は移動解決後の盤面をもとに決定する。
- 射程内に敵が1体しかいない場合、自動選択される。
- 射程内に敵が複数いる場合、自動優先で対象が選ばれる。
- プレイヤーは必要に応じて攻撃対象を変更できる。
- 攻撃対象の表示優先は、王、攻撃成功確率の高い対象、軍師・工兵隊などの特殊駒、HP低下駒、拠点内の対象、距離が近い対象、unitId順を基準にする。
- この並び順はUI上の候補表示順であり、戦闘解決順ではない。

## 8.2 射程

- 射程はチェビシェフ距離で判定する。
- 射程1は斜めを含む周囲8マス。
- 弓兵は射程3。
- 工兵隊は射程5。
- 拠点攻撃時は、拠点全体を大きな1マスとして扱う。

## 8.3 攻撃成功確率

- 1回の攻撃成功で体力を1減らす。
- 体力1の駒は1ダメージを受けると撃破される。
- 王は体力3のため、合計3ダメージを受けると撃破される。
- 忍者見習いの攻撃成功確率は未確定であり、実装上は暫定弱兵種として扱う。

### 鼓舞補正

- 鼓舞補正は攻撃成功確率の分母を1減らす。
- 1/5は1/4、1/6は1/5、1/7は1/6になる。
- 鼓舞補正は重複しない。
- 水面上の駒は鼓舞対象にならない。

### 撤退中歩兵への攻撃

- 攻撃対象が撤退中の歩兵である場合、攻撃側の最終攻撃成功確率を半減する。
- 例: 1/6は1/12、1/5は1/10になる。
- この補正は歩兵の撤退中にのみ適用する。
- 撤退中の歩兵は攻撃できない。

## 8.5 複数攻撃

- 1体の敵に対し、複数の味方駒が攻撃対象を指定できる。
- 拠点攻防戦では集中攻撃が重要な戦術となる。
- 複数攻撃により、低い攻撃成功確率でも撃破確率を高められる。

## 8.6 戦闘解決方針

戦闘解決は以下の方針で行う。

- 移動解決後の盤面を基準にする。
- 保存済みAttackIntentをすべて再検証する。
- 戦闘開始時点で有効なAttackIntentだけをBattleEventへ変換する。
- 各BattleEventの基本攻撃成功確率、鼓舞補正有無、最終攻撃成功確率を決定する。
- GameStateを変更せず、すべてのBattleEventの成否を判定する。
- すべての成否判定終了後、対象ごとに成功数を合算する。
- HP減少、撃破、removed、王撃破、Team.status変更をまとめて反映する。
- 戦闘ログを生成する。

### 完全同時解決

- 戦闘開始時点で有効だった攻撃は、攻撃者が同じ戦闘内で撃破されても発生する。
- 相互攻撃が両方成功した場合、刺し違えによる同時撃破を認める。
- 同じ対象へ複数攻撃が成功した場合、成功数分のダメージを合算する。
- 配列順、unitId順、兵種順による先攻・後攻は作らない。
- 攻撃イベントの解決中にGameStateを逐次変更しない。
- 軍師など攻撃権を持たない駒のAttackIntentは、有効BattleEventに変換しない。

### 廃止案: 攻撃階層

priorityTierによるS/A/B/C階層解決、resolutionLayerによる順次解決、兵種相性による先攻・後攻判定、上位階層で撃破された駒の下位攻撃キャンセルは採用しない。

# ※撤退ルール（暫定）

## 撤退状態の位置づけ

撤退状態は、歩兵専用ではなく全兵種に発生し得る共通状態である。  
ただし、撤退中に防御補正を得るのは歩兵のみである。

このルールは暫定仕様であり、実際のプレイテストで有効性を確認してから最終確定する。

## 撤退状態の共通効果

撤退中の駒には、兵種を問わず以下を適用する。

- 撤退中の駒は攻撃できない。
- 撤退中の駒は攻撃対象になり得る。
- 撤退中の駒が自軍拠点へ到達した場合、拠点内へ入り、撤退状態は解除される。
- 撤退中の駒が敵拠点側へ移動した場合、撤退状態は自動解除される。
- 攻撃対象決定フェーズで撤退中の駒が敵と接している場合、プレイヤーは撤退を維持するか、撤退を解除して攻撃するかを選択できる。

## 撤退状態の発動条件

駒は、以下をすべて満たす場合に撤退中になる。

1. この駒が直前の戦闘で敵拠点から3マス以内の戦闘に参加している。
2. 次の移動入力で、この駒が自軍拠点方向へ移動する。
3. 自軍拠点方向への移動とは、移動後の位置が移動前よりも撤退先自軍拠点に近づくことを意味する。

## 歩兵の撤退補正

歩兵は撤退に適した兵種であり、撤退中に防御補正を得る。

- 通常の攻撃成功確率1/6の攻撃は、撤退中の歩兵に対しては1/12になる。
- 攻撃側に鼓舞などの攻撃成功確率上昇がある場合、最終攻撃成功確率を半減する。
  - 例：1/5 → 1/10
- この防御補正は歩兵にのみ適用される。

## 歩兵以外の撤退

歩兵以外の駒も撤退状態になり得る。  
ただし、歩兵以外の駒は撤退中に防御補正を得ない。

- 騎兵、弓兵、忍者、工兵隊、軍師は撤退中に攻撃できない。
- これらの駒は撤退中でも通常の攻撃成功確率で攻撃を受ける。
- つまり、歩兵以外の撤退は「防御特化」ではなく、「攻撃できない退避状態」である。

# 9. 橋ルール

## 9.1 橋の設置

- 橋は建設軍師が設置する。
- 橋はマップ上で指定された同一湖内の直線区間にのみ設置できる。
- 橋は縦または横一直線のみ。
- 斜め橋は不可。
- 橋の長さは数値上限ではなく、マップ側の橋候補スロットで制御する。

## 9.2 橋の設置範囲

橋は以下の条件を満たす場所に設置できる。

1. 始点側が自軍作戦圏の道マスに接する湖である。
2. 終点側が同じ湖に接する別の道である。
3. 終点側の道が自軍作戦圏である必要はない。
4. 橋はマップ定義済みの橋候補スロットにのみ設置できる。

## 9.3 橋の通行

- 橋は敵味方を問わず通行可能。
- 橋上の駒は攻撃可能。
- 橋上の駒は攻撃対象になる。
- 橋は通常地上マスに近い扱いだが、水計対象となる特殊地形である。

## 9.4 忍者と橋

- 忍者は水面移動中、橋を水面として横切れない。
- 忍者は水面から橋へ上陸できない。
- 忍者が地上側から橋へ通常移動する扱いについては、実装時に明確化する。

## 9.5 橋リセット

- 橋を別の場所へ移動させることは即時にはできない。
- 別の場所に橋を作りたい場合、まず現在の橋をリセット消去する。
- 橋リセット後、橋は5ターンのクールダウンに入る。
- クールダウン終了後、橋は未設置状態となり、新たに設置可能になる。
- 5ターンは初期値であり、調整可能。

## 9.6 橋リセット不可条件

以下の場合、橋リセットは選択不可とする。

- 自軍王が現在の橋上にいる。
- その橋を管理する建設軍師本人が現在の橋上にいる。
- 橋が存在しない。
- 橋がすでにクールダウン中である。

UIでは選択不可としてグレー化する。

## 9.7 水計

- 手動の橋リセット時だけ水計が発生する。
- 忍者は元の湖マスに残る。王以外の非忍者は即時除去される。
- 橋上の敵一般兵が水没した場合、リセットチームの撃破実績に加算する。味方一般兵は損失だけとする。
- 王はHPを1失い、生存時は全盤面の最寄り空き通常道路、所有拠点、死亡の順で退避処理する。敵王へのダメージは王攻略累積ダメージへ加算するが、有効攻撃ターン数には加算しない。
- 自軍王や管理軍師本人が橋上にいる場合、そもそも橋リセットできない。

## 9.8 橋の消滅

橋は管理軍師による手動リセットで消滅する。管理軍師死亡やチーム敗北では消滅せず、管理者不在設備または所有者不在の残留設備として盤面効果を維持する。

# 10. 障害物ルール

## 10.1 障害物の設置

- 障害物は建設軍師が設置する。
- 障害物の管理上限は建設軍師ごとに管理し、正式な征服数に応じて1個または2個とする。
- 障害物は道マスまたは橋上に設置できる。
- 敵の橋上にも障害物を設置できる。
- 水面そのものには障害物を設置できない。
- 拠点内には障害物を設置できない。

## 10.2 設置条件

障害物を設置できるマスは以下を満たす。

1. 敵兵・自兵がいない。
2. 敵拠点・自拠点ではない。
3. 水面上ではない。
4. 既存の障害物がない。
5. 設置可能な道マスまたは橋上である。

## 10.3 障害物の効果

- 障害物は移動を妨げる。
- 障害物は近接攻撃を妨げる。
- 弓兵・工兵隊などの遠距離攻撃は障害物を越えて行える。
- 障害物は自軍にも敵軍にも影響する。

## 10.4 障害物リセット

- 障害物を別の場所へ動かす場合、まず現在の障害物をリセット消去する。
- 障害物リセット後、5ターンのクールダウンに入る。
- 橋のクールダウンと障害物のクールダウンは別管理。

## 10.5 橋上障害物

- 橋上に障害物を置ける。
- 障害物をリセットしても橋は残る。
- 橋が消滅またはリセットされた場合、その橋上の障害物も消滅する。
- 橋上の障害物が敵のものであっても、橋リセット時にまとめて消滅する。

# 11. 軍師アクション処理

## 11.1 処理タイミング

軍師アクションは、他の駒の戦闘解決後に処理する。

処理順は以下。

1. 戦闘解決
2. 軍師アクション処理
3. 手動リセット対象と橋上駒のスナップショット
4. 水計、橋上障害物の強制リセット、撃破実績処理
5. 後処理
6. 次ターンへ

## 11.2 建設アクション

建設軍師は暫定的に1ターンに1つの建設アクションを行える。

対象アクション例：

- 橋を設置する
- 橋をリセットする
- 障害物を設置する
- 障害物をリセットする

# 15. UI/入力に関わるルール

## 15.1 移動候補表示

- 駒を選択すると移動候補が表示される。
- 移動候補はロジック上の2D座標から計算する。
- isometric表示上の座標から移動候補を計算してはならない。

### 撤退中歩兵の攻撃確認

攻撃対象決定フェーズで撤退中の歩兵を選択し、その歩兵が敵と接している場合、以下の確認UIを表示する。

「この歩兵は撤退中です。撤退を解除して攻撃しますか？」

選択肢:

- 防御継続

  - 撤退中を解除する。
  - この歩兵は通常状態として攻撃可能になる。
  - 被攻撃成功確率は通常に戻る。

# 攻略・占領・褒賞（実装仕様）

- 各拠点は所有チームを攻略対象とする独立した攻略期間を持つ。拠点内守備駒は、戦闘開始時に `position.kind === "base"` かつ当該拠点の現所有チーム所属である駒に限る。
- 攻略状態は、チーム別の拠点内守備駒撃破数・有効攻撃ターン数、最後の有効攻撃ターン、守備駒損失、陥落決定候補を保持する。正式に成立した攻撃は命中成否を問わずチーム・拠点ごとに1ターン最大1回を記録する。
- 最後の有効攻撃後10ターン連続で有効攻撃がなければ攻略状態をリセットする。10ターン目に有効攻撃があれば継続する。所有者変更時は即時リセットする。
- 守備駒損失0の全退城、または10ターンリセット後の全退城は単純放棄である。攻撃を受けただけでは戦闘中放棄にならない。単純放棄では所有権を維持し、最初の合法入城時に移転するが、占領褒賞・攻略功労補償はない。
- 現攻略期間に守備駒損失があり、リセット前に残存守備駒が全退城すると戦闘中放棄となる。直前ターンの攻撃有無は問わず、移動解決直後に既存駒の後入城を待たず即時占領する。
- 守備隊全滅では陥落決定戦で最後の守備駒を撃破した陥落決定候補だけを対象とする。単独候補は無条件で占領する。複数候補、または戦闘中放棄の参加チームは、撃破数（最大）→合法経路による最寄り生存駒距離（最小）→有効攻撃ターン数（最大）→注入乱数の順で決める。
- 守備隊全滅と戦闘中放棄では占領チームへ占領褒賞駒1体を対象拠点に付与する。占領チームより撃破数が厳密に多い全チームへ攻略功労補償駒を各1体付与する。同数以下は対象外である。
- 褒賞配置要求はキューで保持し、`capture_reward` は対象拠点固定、`contribution_compensation` は対象チーム所有かつ空き枠のある拠点から選択する。通常生産回数・間隔を消費せず、道路上への救済配置はしない。
- 守備隊全滅は `battle_resolution` 後、戦闘中放棄は `movement_resolution` 後に占領処理を行う。未完了要求があれば `reward_placement` へ入り、全要求の完了または失効まで次の通常フェーズへ進まない。褒賞配置は次の通常生産より先である。

保留事項: 3マス以内の出撃守備兵、拠点消滅・焦土化・道路化と動的道区間結合、王撃破得点、占領得点、道路への補償救済は今回実装しない。

## 王攻略・勢力敗北

- 王Unitごとに王攻略記録を持ち、チーム別の王への累積ダメージと王への有効攻撃ターン数を保持する。正式な攻撃は命中成否を問わず同一王・同一チーム・同一ターン最大1回、成功攻撃は実ダメージ量をすべて累積する。同時オーバーキルも成功チーム全員へ加算する。
- 王攻略記録は無攻撃期間、撤退、王の移動・入城・退城ではリセットしない。王撃破、所有拠点0、その他の正式な勢力敗北時に破棄する。
- 王撃破決定戦で王への攻撃に成功した全チームを王撃破候補とする。単独候補は無条件で征服チームとなる。複数候補は累積ダメージ、有効攻撃ターン数、注入乱数の順で絞る。候補外チームは復活しない。
- 単独王撃破では敗北チームの全残存駒を盤面から除去し、所有全拠点を征服チームへ一括継承する。各継承拠点へ固定配置する王撃破褒賞を1件ずつ生成する。
- 征服チームより対象王への累積ダメージが厳密に多い全チームへ、対象王1人につき王攻略補償を各1件生成する。同数以下と征服チームは対象外である。
- 同一戦闘で異なるチームの王が複数撃破された場合、対象全チームを敗北させ、全残存駒を除去し、所有拠点を中立化する。拠点継承、王撃破褒賞、王攻略補償は発生しない。
- 自軍所有拠点内の王は拠点内守備駒でもあるため、王攻略記録と拠点攻略記録の両方を更新する。
- 王撃破と同じ敗北チームの拠点陥落が同じ戦闘で成立した場合は王撃破による一括継承を優先する。本来の占領チームが征服チームと異なる場合、占領褒賞を自軍拠点配置型の占領褒賞変換補償へ変換する。本来の攻略功労補償は維持する。同一チームなら変換補償を重複生成しない。
- 王が生存していても所有拠点0になったチームは敗北し、全残存駒と王攻略記録を除去する。この場合、成立済みの通常占領褒賞と攻略功労補償は維持するが、王撃破褒賞・王攻略補償・一括継承は発生しない。
- 報酬種別は `king_conquest_reward`、`king_contribution_compensation`、`overridden_capture_compensation`、`capture_reward`、`contribution_compensation` を独立要求としてキュー管理する。

王撃破得点、拠点占領得点、最終勝利条件、複数王同時撃破時の自動継承、王攻略記録の無攻撃リセットは保留とする。
# Phase 3-B 撤退経路・防御補正（2026-07-14）

- 正式な撤退には、駒の現在位置から現在所有する自軍拠点まで、既存の道路区間・拠点接続・移動規則に従う合法な退路が必要である。座標距離が短くなるだけでは撤退にならない。
- 敵または中立拠点は撤退経路の中間地点として通過できない。退路がない駒には撤退選択を表示せず、孤立・包囲等の新状態も付与しない。その駒は通常移動・通常攻撃を行える。
- 戦闘参加と敵拠点距離は戦闘開始時点の情報で記録し、最終的な生存、チーム状態、拠点所有権、合法退路、退路方向の最初の一歩は全戦闘・占領・敗北処理後に再検証する。
- 死亡した駒および敗北チームには撤退資格・撤退中状態を残さない。同時戦闘で攻撃者が死亡しても、生存した対象者の参加記録は保持する。
- 正式に撤退中の駒は攻撃Intentを保存できない。撤退候補の未選択、継戦選択、退路なしの通常駒は攻撃可能である。
- 正式に撤退中の歩兵が防御対象の場合だけ、兵種相性と軍師鼓舞を含む既存の最終被弾確率をさらに1/2にする。通常の1/5・1/6・1/7はそれぞれ1/10・1/12・1/14となる。撤退候補だけの歩兵、継戦中、撤退終了後、他兵種には適用しない。
- 正式な撤退開始時に撤退目標拠点IDを固定する。占領、王撃破、中立化、所有拠点0判定等の全解決後、目標の現在所有権と合法経路を再検証する。目標が失効した場合は撤退状態・目標・資格を解除して通常状態へ戻し、別の到達可能な自軍拠点へ自動変更しない。
- 戦闘開始時に正式な撤退中だった駒の攻撃禁止と歩兵防御補正は戦闘開始時スナップショットを使用する。同じ戦闘で目標が失効しても当該戦闘へ遡って補正を取り消さず、全解決後かつ次の入力フェーズ前に撤退状態を解除する。
# Phase 4-A 建設軍師（2026-07-15）

- 戦闘、占領、敗北、全褒賞配置後に `strategist_action_input` を開始し、全activeチーム提出後に `strategist_action_resolution` で同時解決する。褒賞で配置された建設軍師も同ターンから対象になる。
- 生存している軍師は各チーム最大2人。通常生産と褒賞配置の双方で同じ上限を適用する。
- 建設軍師1人はactiveな橋1本と障害物1個を別枠で管理し、1フェーズにつき設置・リセット・何もしないのいずれか1アクションだけを行う。
- 作戦圏は現在所有する自軍拠点に接続する既存道路区間と、その道路に面する湖・接続済みactive橋である。橋の所有チームは問わず、橋の対岸にある別道路区間は橋の存在だけでは作戦圏へ追加しない。障害物は作戦圏自体を分断しない。
- 橋は作戦圏内道路から同一湖の連続湖面を縦横一直線に渡り、対岸道路へ接続する。斜め、曲折、重複、交差、橋からの延長は禁止する。
- activeな橋は所有者を問わず通常道路として移動・攻撃用道路区間を動的に接続する。静的 `roadSectionId` は変更しない。
- 障害物は作戦圏内道路または作戦圏道路に接続した成立済み橋へ設置でき、全チームの移動と撤退経路を遮断する。攻撃接続、射線、命中率には影響しない。
- 同一チームの予定重複は候補生成時に除外する。異なるチームの同種設置が1タイルでも競合した場合、関係する全設置を不成立とし、再入力やクールタイムは発生させない。
- 解決は全リセット、前提再検証、競合除外、非競合設置の順。新設予定橋へ同じフェーズで障害物を予約できない。
- 橋・障害物のリセット後は設備種別ごとに `availableFromTurn = resetTurn + 5` を保持する。T+4は不可、T+5から再設置可能。
- 自チームの設置予定だけを薄いグレーで表示し、相手の未解決Intentは公開しない。
- CPU実装前のデバッグ画面では操作対象のactiveチームを切り替え、選択チームの所有拠点、生産候補、建設軍師、保存済みIntentおよび設置予定だけを表示する。全チームの提出済み状態を明示する。
- 作戦圏は各所有拠点に接続する全道路区間の和集合とする。4人用初期マップのTeam 1本拠地では北・西・中央方向の3区間すべてを起点にし、敵・中立拠点の反対側へは拡張しない。
- 橋候補は全作戦圏道路と全直交方向を走査し、両岸から得られる逆順表現を同一橋として正規化する。基準盤面のTeam 1候補は障害物27か所、橋28通りである。
- 橋上駒の水計、王撃破帰属、管理軍師死亡時の設備残留、設備継承、征服数による管理上限はPhase 4-B確定仕様に従う。
# Phase 4-B 橋リセット時の水計・設備管理権（2026-07-18確定）

## 手動橋リセットと水計

- 水計は、管理中の橋を建設軍師アクションで手動リセットした場合に限り発生する。建設軍師死亡、チーム敗北、設備所有権移転では発生しない。
- リセット対象の橋上に、その橋の管理建設軍師またはリセットチーム自身の王がいる場合、手動リセットは不成立とする。それ以外の味方・敵の一般兵、敵王、他の建設軍師がいてもリセットできる。
- 同時に複数橋をリセットする場合、消滅直前の全橋と橋上駒を先にスナップショットし、同じ駒を二重処理しない。
- 忍者は橋の元湖マスに `water` 状態で残り、ダメージ、撃破、退避、貢献記録を発生させない。
- 王以外の非忍者は即時除去する。敵駒はリセットチームの `defeatedUnitCount` に加算し、味方駒は損失だけとする。拠点攻略・王攻略の貢献値には加算しない。
- 王はHPを1失う。敵王への1ダメージは王攻略の累積ダメージへ加算するが、有効攻撃ターン数には加算しない。HP0または退避不能死亡は既存の王撃破・チーム敗北処理へ接続し、敵王についてはリセットチームを撃破候補とする。

## 水計後の王の退避

- 生存王の所属チームの作戦圏、道路所有、`roadSectionId`、拠点との接続を問わず、盤面全体の通常道路を第1候補とする。駒、activeな障害物、activeな橋が存在するタイル、拠点・湖タイルは候補外とする。駒が水没した元橋タイルからの盤面距離が最小の候補を選び、同距離候補は座標で正規化してから注入乱数で決める。
- 複数対象は道路退避をまとめて解決し、同じ道路マスへ重複配置しない。
- 空き通常道路が盤面全体に1件もない場合に限り、所属チーム所有拠点の空きBaseSlotへ強制帰還する。元橋タイルから最寄りの拠点を優先し、同距離に本拠地があれば本拠地を優先し、それでも同条件なら安定キーで正規化してから注入乱数で決める。同じBaseSlotへ重複配置しない。
- 道路にも自軍拠点にも空きがない場合は死亡とする。敵ならリセットチームの撃破実績へ加算し、味方なら損失だけとする。

## 橋上障害物

- 手動リセットされた橋上のactive障害物は所有チームを問わず全て強制リセットする。
- 橋管理軍師の橋枠と、橋上障害物に管理軍師がいる場合の障害物枠は、それぞれ `availableFromTurn = currentTurn + 5` とする。管理者不在設備にはクールタイムを設定しない。
- 強制リセットは障害物管理軍師の追加アクションを消費しない。解決順は既存どおり、リセット、水計と強制リセット、設置Intent再検証、競合解決、設置とする。

## 管理者不在設備と継承

- 建設軍師死亡時、その軍師が管理していた橋・障害物はactiveのまま残し、所有チームを維持して管理者だけを未設定にする。死亡による水計は発生しない。
- 管理者不在設備は盤面効果を維持するが、管理者が割り当てられるまでリセット・再配置できない。
- 同じ所有チームの後継建設軍師へ、管理上限内で管理権を明示的に割り当てられる。上限超過設備は削除せず管理者不在のまま残す。
- 単独王撃破による正式な征服では、敗北チーム設備の所有チームを征服チームへ変更し、管理者を未設定にする。複数王同時撃破による中立化では、敗北チーム設備を所有者・管理者とも未設定の残留物とし、誰にも継承させない。

## 征服数と設備管理上限

- 建設軍師は最大2人、1軍師1ターン1アクションを維持する。
- 正式な征服数0では各建設軍師が橋1・障害物1を管理できる。
- 正式な征服数1では、選択した片方の建設軍師が橋2・障害物2、もう片方が橋1・障害物1を管理できる。強化先死亡時は未割当へ戻し、後継へ再割当できる。
- 正式な征服数2以上では両建設軍師が橋2・障害物2を管理できる。各種2件を上限とする。
- 複数王同時撃破による中立化は正式な征服数へ加算しない。
